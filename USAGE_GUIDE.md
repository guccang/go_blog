# Go Blog 使用指南

## 📖 概述

Go Blog 是一个功能丰富的个人数字生活管理系统，基于创新的"一切皆博客"设计理念。它不仅仅是一个博客系统，更是一个集成了任务管理、锻炼记录、阅读管理、AI助手等功能的综合性平台。

### 核心特点
- **统一数据模型**：所有数据都以Markdown博客格式存储
- **模块化架构**：20+独立功能模块，支持灵活扩展
- **多用户支持**：每个用户拥有独立的数据空间
- **AI集成**：内置DeepSeek AI助手，支持智能分析
- **数据自主**：所有数据均为文本文件，用户完全掌控

## 🚀 快速入门

### 环境要求
- **Go 1.21+**：Go语言运行环境
- **Redis**：用于缓存和会话管理
- **Git**：版本控制（可选）

### 安装步骤

#### 1. 获取项目
```bash
git clone <repository_url>
cd go_blog
```

#### 2. 安装依赖
```bash
go mod tidy
```

#### 3. 配置系统
编辑系统配置文件 `blogs_txt/sys_conf.md`：
```markdown
port=8888
redis_ip=127.0.0.1
redis_port=6666
admin=your_username
pwd=your_password
```

#### 4. 启动Redis
```bash
./scripts/start_redis.sh
```

#### 5. 编译和运行
```bash
# 编译项目
./scripts/build.sh

# 启动服务
./scripts/start.sh
```

#### 6. 访问系统
打开浏览器访问：`http://localhost:8888`

## ✍️ 博客功能使用指南

### 创建新博客
1. 点击导航栏的"新建博客"按钮
2. 输入博客标题和内容
3. 设置权限标签（public/private/encrypt/diary）
4. 点击保存

### 权限类型说明
- **public**：公开博客，所有访客可见
- **private**：私有博客，仅自己可见
- **encrypt**：加密博客，需要密码访问
- **diary**：日记博客，额外密码保护

### Markdown编辑器功能
- 实时预览模式
- 语法高亮支持
- 图片上传和插入
- 表格创建工具
- 代码块支持

### 博客管理
- **标签分类**：为博客添加标签便于查找
- **全文搜索**：支持标题和内容搜索
- **批量操作**：支持批量删除、导出
- **分享功能**：生成有时效性的分享链接

## 🏋️ 锻炼管理系统

### 功能概述
锻炼管理系统支持四种锻炼类型：
1. **有氧运动**：跑步、游泳、骑行等
2. **力量训练**：举重、器械训练等
3. **柔韧训练**：瑜伽、拉伸等
4. **体育运动**：篮球、足球、羽毛球等

### 智能计算功能
系统基于MET值（代谢当量）精确计算卡路里消耗：
- 根据用户体重和锻炼时长自动计算
- 支持自定义MET值设置
- 历史数据趋势分析

### 使用指南

#### 1. 添加锻炼记录
- 选择锻炼类型
- 输入锻炼项目和时长
- 系统自动计算卡路里消耗
- 可添加详细备注

#### 2. 模板管理
- 创建常用锻炼模板
- 支持模板组合（集合）
- 一键添加整套训练计划

#### 3. 个人档案
- 设置身高、体重等基本信息
- 查看个人锻炼统计数据
- 设置健身目标

#### 4. 数据可视化
- 周/月/年度锻炼统计图表
- 卡路里消耗趋势图
- 锻炼类型分布图

## 🤖 AI助手使用指南

### 功能概述
Go Blog集成了DeepSeek AI助手，可以：
- 分析博客内容提供建议
- 协助内容创作和编辑
- 提供个性化推荐
- 通过MCP协议访问系统数据

### 配置AI助手
1. 获取DeepSeek API密钥
2. 在 `sys_conf.md` 中配置：
   ```markdown
   deepseek_api_key=sk-your-api-key
   deepseek_api_url=https://api.deepseek.com/chat/completions
   ```

### 使用场景

#### 1. 内容分析
- 上传博客内容让AI分析
- 获得改进建议和优化意见
- 检查语法和逻辑错误

#### 2. 创作协助
- 根据主题生成内容大纲
- 协助编写技术文档
- 提供创意灵感

#### 3. 数据洞察
- 分析锻炼数据提供建议
- 解读阅读习惯
- 任务完成情况分析

#### 4. MCP集成
- AI可以通过MCP协议访问：
  - Redis缓存数据
  - 本地博客文件
  - 系统配置信息

## 💾 数据管理

### 数据存储结构
```
blogs_txt/
├── sys_conf.md          # 系统配置
├── hjy/                 # 用户hjy的数据目录
├── online/             # 用户online的数据目录
└── ztt/                # 用户ztt的数据目录
```

### 备份和恢复

#### 1. 数据备份
```bash
# 备份整个数据目录
cp -r blogs_txt/ blogs_txt_backup_$(date +%Y%m%d)
```

#### 2. 数据恢复
```bash
# 恢复数据目录
cp -r blogs_txt_backup/ blogs_txt/
```

#### 3. 导出功能
- 支持单个博客导出为Markdown文件
- 批量导出为ZIP压缩包
- 导出统计数据为CSV格式

### 数据迁移
由于所有数据都是Markdown文件，迁移非常简单：
1. 复制 `blogs_txt/` 目录到新环境
2. 配置相同的系统设置
3. 启动服务即可

## 🔧 故障排除

### 常见问题

#### 1. 服务无法启动
- 检查端口是否被占用：`lsof -i :8888`
- 验证Redis是否运行：`redis-cli -p 6666 ping`
- 检查配置文件语法

#### 2. 页面访问错误
- 确认服务已成功启动
- 检查防火墙设置
- 验证浏览器兼容性

#### 3. 数据不显示
- 检查数据目录权限
- 确认Redis连接正常
- 查看应用日志：`logs/blog_*.log`

#### 4. AI功能不可用
- 验证API密钥是否正确
- 检查网络连接
- 确认API配额充足

### 日志查看
```bash
# 查看应用日志
tail -f logs/blog_*.log

# 查看访问日志
tail -f logs/access.log
```

### 性能优化建议
1. **Redis配置**：根据数据量调整内存限制
2. **文件缓存**：启用浏览器缓存静态资源
3. **数据库优化**：定期清理过期缓存数据
4. **CDN加速**：生产环境使用CDN加速静态资源

## 🔐 安全建议

### 账户安全
1. **使用强密码**：包含大小写字母、数字、特殊字符
2. **定期更换密码**：建议每3个月更换一次
3. **避免共享账户**：每个用户使用独立账户

### 数据安全
1. **定期备份**：重要数据定期备份到外部存储
2. **加密敏感内容**：使用encrypt权限保护隐私
3. **访问控制**：合理设置博客权限

### 网络安全
1. **启用HTTPS**：生产环境必须使用HTTPS
2. **限制访问IP**：配置防火墙规则
3. **定期更新**：保持系统和依赖包最新

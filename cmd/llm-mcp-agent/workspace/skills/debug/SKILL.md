# 问题排查

## 排查流程

1. **收集信息**：先用 ListLogSources 查看可用日志源列表
2. **查看日志**：用 ReadLog 读取相关日志，关注错误信息和异常堆栈
3. **分析数据**：如需批量分析日志数据，使用 ExecuteCode 沙箱处理
4. **定位问题**：根据日志信息定位根因
5. **决定方案**：根据问题类型决定修复方案（修改代码/调整配置/重启服务）

## 注意事项

- 先查日志再下结论，不要猜测问题原因
- 日志量大时使用 ExecuteCode 做数据分析和过滤
- 定位问题后，根据情况决定是否需要编码修复

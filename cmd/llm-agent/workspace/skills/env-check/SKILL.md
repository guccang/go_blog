# 环境检测

## 适用场景

- 用户需要确认服务器或本地环境是否满足运行条件
- 需要检查软件版本、磁盘、内存、进程、端口、系统信息
- 需要在部署前或排障时快速给出环境快照

## 必须遵守

- 先确认检查目标是本地还是远程，以及要验证的软件或服务
- 远程检查使用上下文里的真实目标地址，不能把固定主机写死在通用流程里
- 检测结果要明确标出异常项，而不是只贴原始命令输出
- 用户只问某一项时，只检查必要内容，不要无关扩展

## 推荐流程

1. 确认目标机器、要检查的软件或服务，以及期望状态。
2. 执行最相关的检查命令，例如：
   - Go 版本：`go version` 或 `ssh <user>@<host> 'go version'`
   - Node 版本：`ssh <user>@<host> 'node --version'`
   - 磁盘空间：`ssh <user>@<host> 'df -h'`
   - 内存使用：`ssh <user>@<host> 'free -h'`
   - 进程状态：`ssh <user>@<host> 'ps aux | grep <process>'`
   - 端口监听：`ssh <user>@<host> 'ss -tlnp | grep <port>'`
3. 对结果做摘要，标出版本是否符合预期、资源是否紧张、进程或端口是否异常。

## 工具选择规则

- 统一使用 `Bash` 执行本地或远程环境检查命令
- 单台远程服务器检查时，用 `ssh <user>@<host>` 包装命令即可
- 如果用户只问版本或端口，优先执行最小命令集，不要扩大检查范围

## 禁止行为

- 在 skill 中写死固定 IP、固定账号或某个特定服务器
- 不做整理，直接把大段命令输出原样堆给用户
- 用户只问一个问题时，顺手做一整套无关巡检
- 对未执行的检查项给出“应该正常”的推测结论

## 示例

- “帮我确认生产机有没有安装 Go”
  先确认目标主机，再执行 `go version`
- “检查 8080 端口是不是被占了”
  直接查端口监听和相关进程，再汇报占用者
